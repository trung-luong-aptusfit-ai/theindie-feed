In the blog post titled Just Talk to It: The No-BS Way of Agentic Engineering, there is a graphic illustrating the curve of agentic programming. While some call this vibe coding, that term is often used as a slur. A better term is agentic engineering. The Composability of CLIs vs. MCPs

[2:41:07] A significant advantage of models is their inherent ability to call Unix commands. A CLI is ultimately just another Unix command, which is a very natural interface for a model to interact with. In contrast, the Model Context Protocol (MCP) often requires specific syntax that must be addressed during model training, making it less intuitive.

The most substantial issue with MCP is that it is not composable. For example, if a service provides a large data blob containing temperature, average temperature, rain, wind, and other metrics, the model is forced to ingest that entire "huge blob" into its context before it can pick out the specific information it needs. There is no natural way for the model to filter that data unless the developer proactively builds filtering logic into the MCP itself.

If the same service were built as a CLI, the model could simply add a JQ command to filter the output and retrieve only the necessary data. It could even compose a script to perform calculations—such as processing temperature data—and return only the exact result. This approach prevents context pollution.
